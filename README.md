## ⚙️ Архитектура данных и автоматизация (Data Pipeline)

В проекте реализована гибридная схема обновления данных, сочетающая ручной контроль и облачную автоматизацию через **GitHub Actions**:

1.  **Первичная загрузка (Manual Ingestion):** 
    *   Сбор сырых данных ГИБДД через Jupyter Notebook [`data_loading_api.ipynb`](./data_loading_api.ipynb).
    *   Данные временно сохраняются локально, проходят проверку и через **GitHub Desktop** загружаются в репозиторий в папку.
    *   *Примечание:* После пуша в репозиторий локальные файлы можно удалять — GitHub выступает в роли основного облачного буфера и архива.

2.  **Синхронизация с БД (GIBDD Sync):**
    *   GitHub Action раз в месяц считывает новые JSON-файлы из архива репозитория и автоматически загружает их в **Supabase** для обновления дашборда.

3.  **Ежедневные обновления (Weather Update):**
    *   Автоматизированный скрипт каждое утро в 09:00 (МСК) собирает актуальные метеоданные через API и записывает их в базу данных.

4.  **Справочники (City Reference):**
    *   Дважды в год скрипт актуализирует геоданные и справочники городов через API Dadata и Geopy.

## 📁 Структура репозитория
*   `data_loading_api.ipynb` — Ноутбук для сбора данных и первичной обработки.
*   `init_db.sql` —  Скрипт инициализации структуры базы данных (DDL), создание таблиц, индексов и связей.
*   `data_analysis.ipynb` — Исследовательский анализ данных (EDA).
*   `dashboard_screenshots` — Скриншоты дашборда.
*   `scriptsupdate_data.ipynb` — Конфигурации автоматических запусков (CI/CD) и Python-скрипты для обновления погоды и синхронизации с базой.
